aSOLNP+ User GuideSOLNP+ Team: Jiyuan Tan, Tianhao Liu, Jinsong LiuSection 1 Brief Introduction SOLNP+ is a derivative-free solver proposed by Professor Yinyu Ye for general nonlinear constrained problems in 1989 (Origin name SOLNP), which was first implemented in Matlab. It solves the following problem,
minx f(x)
s.t. g(x)=0,
lhh(x)uh,
lxxux,
minxf(x)s.t.g(x)=0,lhh(x)uh,lxxux,where f:RnR,g:RnRm1,h:RnRm2f:RnR,g:RnRm1,h:RnRm2 are smooth functions. The solver is designed for black-box optimization, where the user cannot provide an explicit analytic expression. We suggest that users should not use SOLNP+ for problems with explicit formulas as other (first-order or second-order) methods may be more efficient than the zero-order methods. SOLNP+ is suitable for the following situations. The user only has access to the value of the functions. There are some noises when calculating the functions. Based on the original code of Professor Ye, SOLNP+ is implemented by the SOLNP+ team in ANSI C with some improvements. Currently, SOLNP+ only provides Matlab interface.Section 2 Installation(a) Prerequisites Before installation, the user should install the Matlab software in his computer. Besides, a supported C/C++ compiler should be installed. See compiler for supported C/C++ compiler in Matlab. Besides, you need to install MKL and OSQP in your computer. SOLNP+ currently support Window system with msvc compiler.(b) Download The user can download the source in SOLNP+.(c) Installation After downloading the files, the user can find make_SOLNP+.m in the dictionary. Change the path of OSQP and MKL in make_solnp.m.
Run make_solnp.m with the Matlab software. If the command box shows
MEX completed successfully.
it means SOLNP+ has been successfully installed. The user should add path WORKING to Matlab path, where WORKING is the dictionary where the user download the source file.Section 3 SOLNP+ Usage (a) Function File The user should provide a .m file called cost.m that returns the objective and constraint function value at given point pp. The first line of the file is the MATLAB function declaration.
function f = cost(p)
pp is the input variable used to evaluate the function values. The output ff is a vector. f(1)f(1) is the value of the objective function evaluated at xx. f(2)f(2) through f(m1+1)f(m1+1) are the values of the equality constraints, and f(m1+2)f(m1+2) through f(m1+m2+1)f(m1+m2+1) are the values of the inequality constraints. The order of the inequality constraints should be consistent with the input.(b) Input The user can use the following command to call SOLNP+.
info = SOLNP_plus(prob, options);
where prob and options are structures that contains the information of the problem and parameters of the algorithm. The fields of prob are prob.pbl is the lower bound of the box constraints (lxlx in (3)). The default setting is --. prob.pbu is the upper bound of the box constraints (uxux in (3)).The default setting is ++. prob.ibl is the lower bound of the inequality constraints (lhlh in (3)). The default setting is --. prob.ibu is the upper bound of the inequality constraints (uhuh in (3)).The default setting is ++. prob.p0 is the initiate point of the algorithm. It should be within the interior of the box constraints. If the user does not provide initiate point and both prob.pbl and prob.pbu are finite, the default initiate point is set to be the center point of the box, i.e., (prob.pbl + prob.pbu)/2. prob.ib0 is the estimated inequality value of the inequality constraints. It should be within the interior of the inequality constraints. If the user does not provide the estimation and both prob.ibl and prob.ibu are finite, the default estimation is set to be the center point of the box, i.e., (prob.ibl + prob.ibu)/2.For example, if we want to solve a equality constrained problem with initiate point (1,1,2,3)(1,1,2,3), we may set
prob.p0 = [1,1,2,3]';
If we want to solve a problem with inequality constraints and box constraints, we may set
prob.pbl = l_x; prob.pbu = u_x;prob.ibl = l_h;prob.ibu = u_h;
options contains the parameter of the algorithm. The fields of options are options.tol: Convergence tolerance. The algorithm will stop when the relative difference is less than options.tol. Default value 10-410-4. options.tol_con: Tolerance of infeasibility. Default value 10-310-3. options.tol_restart: Tolerance of restart. SOLNP+ uses gradient of augmented Lagrange function to decide whether to restart. Mind that due to limitation of zero-order method, the gradient may be large even the point is close to the optimal point. Default value 11. options.ls_time: Maximum number of function evaluations when performing line search each time. Default value 1010. options.re_time: Maximum number of restart time. Default value 55. options.maxfev: Maximum number of function evaluations. Default value 500n500n. nn is the dimension of the input pp. options.max_iter: Maximum number of outer iteration. Default value 5050. options.min_iter: Maximum number of inner iteration. Default value 1010. options.noise: whether there are noises when evaluating function value. options.noise = 11 means there are noises and the algorithm will choose to use adaptive delta when computing approximated gradient. options.noise = 00 means there are no noise and the algorithm will use fixed delta. Default value 1.1. options.rho: The initiate penalty parameter of augmented Lagrange function. Default value 1.1. options.delta: The initiate delta when computing approximated gradient. Default value: 11 if options.noise = 11, 10-510-5 if options.noise = 00. options.delta_end: The lower bound of delta. If the user chooses to use adaptive delta, delta will not be lower than options.delta_end during calculation. Default value 10-510-5. options.k_i: The expand ratio of step size : k+1=kik.k+1=kik. It will increase more quickly with a larger kiki value. Default value ki=3ki=3. options.k_r: The reduce ratio parameter of step size : k+1=k/kr.k+1=k/kr. It will reduce more quickly with a larger krkr value. Default value kr=9kr=9. options.c_i: Used for determine when the step size should increase. It will increase more quickly with a smaller cici value. Default value ci=30ci=30. options.c_r: Used for determine when the step size should reduce. It will reduce more quickly with a larger crcr value. Default value cr=10.cr=10. options.qpsolver: Default value 11. options.qpsolver = 1: Use interior method to solve QP subproblem. options.qpsolver = 1: Use OSQP to solve QP subproblem.(c) Output The output structure has the following fields output.p: The primal solution. output.obj: Objective function value. output.l: The dual solution. output.h: The estimated Hessian matrix. output.jh: History of primal value. output.jh: History of infeasibility. output.iter: Number of outer iteration. output.count_cost: Number of function evaluations. output.constraint: Infeasibility. output.ic: Estimated inequality constraints value. output.status: Status of the algorithm output.status = 1: Success output.status = 0: Exiting after maximum number of function evaluation. output.status = -1: Exiting after maximum number of iterations. Tolerance not achieved. output.status = -2: Exiting after maximum number of iterations. Tolerance of infeasibility achieved. The algorithm fails to converge. output.status = -3: Exiting because of unknown numerical error.(d) Examples The user can find some test examples in the dictionary ./test .(e) Result Analysis After the algorithm has completed the iterative process, you should be careful to check the final solution to see if it is feasible and optimal by common sense. There is no guarantee that the final solution is the global solution. Restart SOLNP+ if it terminates at the maximum number of major iterations.
